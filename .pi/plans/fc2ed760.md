## Context

Current `brave-search` skill can't fetch login-walled pages or JS-rendered content. Need a new `browser-fetch` skill that fills both gaps using Playwright + Firefox cookie injection.

## Design Decisions

- **Single Playwright code path** (no fetch-first/fallback hybrid). Always renders JS, always injects cookies. 3-5s per page is fine for the use case.
- **Firefox cookie extraction** via `sqlite3` CLI (ships with macOS). No native Node deps. Copy `cookies.sqlite` first since Firefox locks it. Use unique temp path, clean up after.
- **Auto-detect Firefox profile** by parsing `profiles.ini` — use the `[Install*]` section's `Default` key (NOT the profile with `Default=1` flag, which is different).
- **Must download Playwright Chromium** — user has no system Chrome. ~200MB one-time cost. Setup step required.
- **Output format** matches brave-search: markdown to stdout.
- **Known limitations**: localStorage-based auth (Discord, some SPAs) won't work with cookie injection alone. Document in SKILL.md.

**fetch.js workflow:**
1. Parse target URL → extract domain
2. Copy Firefox `cookies.sqlite` to unique temp path → query for matching domain cookies via `sqlite3` CLI → filter expired (`WHERE expiry = 0 OR expiry > unixepoch()`) → clean up temp file
3. Convert cookie format: Firefox → Playwright (sameSite 0/1/2 → None/Lax/Strict; isSecure/isHttpOnly int → bool; host with leading dot → domain; expiry 0 = session cookie → omit expires)
4. Launch Playwright headless Chromium (downloaded, not system Chrome — user has no Chrome)
5. Create context → inject cookies via `context.addCookies()`
6. Navigate to URL with `{ waitUntil: 'load', timeout: 15000 }` (NOT networkidle — hangs on websocket/polling pages)
7. Optional: small delay (1-2s) for JS frameworks to finish rendering
8. Extract HTML via `page.content()` → Readability → Turndown → markdown to stdout
9. Close browser

**Cookie domain matching logic:**
- URL `https://sub.example.com/path` should match cookies for:
  - `.example.com` (leading dot = all subdomains)
  - `sub.example.com` (exact match)
  - `.sub.example.com` (if exists)
- SQL: `WHERE host = :domain OR host = '.' || :domain OR :domain LIKE '%' || host`
