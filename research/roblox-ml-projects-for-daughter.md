## Key Research Findings (March 2026)

### Roblox Has Native AI APIs Now

As of mid-2025, Roblox ships built-in AI:

- **TextGenerator** — built-in LLM via Luau. No external server, no API key, no Flask, free. System prompts supported. 20-message history, **5K token input (~15-20K chars)**, 1K token output. 100 req/sec/experience rate limit.
- **Text-to-Speech** — 10 English voice presets, fully released Oct 2025
- **Speech-to-Text** — beta released Nov 2025, **fully released Feb 2026**. Source: [DevForum full release announcement](https://devforum.roblox.com/t/full-release-speech-to-text-api/4357786)
- **4D Generation** (Feb 2026) — text prompt → fully functional interactive 3D objects (drivable cars, weapons that fire). Uses Roblox's open-source Cube Foundation Model (weights: `Roblox/cube3d-v0.5` on HuggingFace, code: [github.com/Roblox/cube](https://github.com/Roblox/cube)). Currently limited to vehicles and single-mesh objects. Source: [Indian Express, Feb 6 2026](https://indianexpress.com); [battleofguardians.com, Mar 5 2026](https://battleofguardians.com/roblox-ai-4d-generation-objects-studio-tools-2026/)
- Moderated by **Roblox Guard 1.0** — fine-tuned Llama-3.1-8B-Instruct for safety classification. Open-sourced Jul 2025: model weights on HuggingFace ([`Roblox/Llama-3.1-8B-Instruct-RobloxGuard-1.0`](https://huggingface.co/Roblox/Llama-3.1-8B-Instruct-RobloxGuard-1.0)), eval dataset on HuggingFace ([`Roblox/RobloxGuard-Eval`](https://huggingface.co/datasets/Roblox/RobloxGuard-Eval)), eval framework on GitHub ([`Roblox/RobloxGuard-1.0`](https://github.com/Roblox/RobloxGuard-1.0)). Source: [Roblox Newsroom, Jul 2025](https://about.roblox.com/newsroom/2025/07/roblox-guard-advancing-safety-for-llms-with-robust-guardrails)

```lua
local tg = Instance.new("TextGenerator", workspace)
tg.SystemPrompt = "You are a mysterious oracle who speaks in riddles."

local success, response = pcall(function()
    return tg:GenerateTextAsync({ UserPrompt = "Where is the hidden treasure?" })
end)

if success then
    print(response.GeneratedText)
else
    warn("Text generation failed: " .. tostring(response))
end
```

### ⚠️ Critical Constraint: "Extended Interactions" Forces 17+ Rating

As of September 2025, Roblox expanded TextGenerator access to all maturity ratings **except experiences featuring "extended interactions."** Prolonged LLM conversations (more than a few exchanges) force the game to **Restricted (17+)** content maturity. Source: [DevForum: Text Generation API Beta Expanding Access](https://devforum.roblox.com/t/text-generation-api-beta-expanding-access/3952348); [robloxdb.com Sep 2025 roundup](https://robloxdb.com/blog/post/roblox-september-2025-updates-lighting-ai)

**Impact:** 17+ games require government ID age verification. A 10th grader's friends (~15-16yo) generally cannot play. The DevForum community is vocal about this — multiple developers report it caps active players at ~30 and kills adoption.

**Mitigations:**
- Design NPCs for short interactions only (≤3 exchanges) — a genuine design challenge: make an NPC feel alive in 2-3 sentences
- Route LLM calls through HttpService to an external API (avoids native API restriction but adds server-side complexity)
- Accept the 17+ restriction and design for a smaller audience
- Use native API for development/prototyping, plan HttpService backend for distribution

This constraint reshapes Projects #1 and #2 significantly. Any Roblox project using TextGenerator for extended NPC conversations must account for it.

### 🏆 1. Backprop from Scratch with Calc BC (Mandatory First Project)

Implement a tiny neural net in raw NumPy using chain rule from Calc BC. 2-layer net, sigmoid, manual gradients, train on MNIST.

**The payoff:** She'll viscerally understand that backprop IS the chain rule — the same math she's doing in class right now, applied to the thing she's curious about. Every other project on this list becomes deeper once she has this mental model.

### 🔥 2. Evaluate Roblox Guard: Open-Source AI Safety Research

Roblox Guard 1.0's model weights and eval framework are **open-sourced**. She can do rigorous ML evaluation in Python — her strongest language — with no ToS risk.

**The project:**
- Download the model (`Roblox/Llama-3.1-8B-Instruct-RobloxGuard-1.0`) and run it locally
- Evaluate against Roblox's own benchmark (`Roblox/RobloxGuard-Eval`) — reproduce their SOTA claims
- Build custom test cases informed by her domain expertise: Roblox-native slang, coded language, what kids actually try to say. This is exactly the context most ML safety researchers lack.
- Compare against Llama Guard (Meta), ShieldGemma (Google), GPT-4o — Roblox claims to outperform all of them. Verify it.
- Build a taxonomy of failure modes: false positives (safe content blocked), false negatives (unsafe content passed), edge cases
- Write it up as a proper evaluation paper

**ML depth:** Model evaluation, classification metrics (precision/recall/F1), dataset curation, benchmark design, safety taxonomies. Real ML skills, not just prompt engineering.

**Why this fits her:** She can do rigorous ML evaluation in Python on an open-source model with real-world impact. ~~However, the "Roblox domain expert" angle has limits:~~ she's a solo/casual player, not active in Roblox's social/chat layer. She doesn't walk in with insider knowledge of evasion tactics, coded slang, or filter bypass patterns. **The custom adversarial test set is still achievable** — but it requires deliberate research (DevForum complaint threads about false positives, documented scam/grooming patterns, Roblox community Discord analysis) rather than lived experience. This is a research task she'd need to scope and budget time for.

**Alternatively, the project stands strong without the domain-expert angle:** reproduce Roblox's SOTA claims, run systematic comparative evaluation across models (RoGuard vs Llama Guard vs ShieldGemma vs GPT-4o), probe for category-level weaknesses in the safety taxonomy (which of the 25 subcategories is weakest?), and analyze where the models disagree. That's a clean, rigorous ML eval paper.

**Science fair angle:** Strong candidate for GSDSEF 2027 → ISEF. "Software Design" (SFTD) category. Clear research question ("Does Roblox Guard 1.0 generalize across safety taxonomy subcategories, and where do leading guardrail models diverge?"), quantitative metrics, reproducible methodology. Can extend with additional models/benchmarks for STS senior year.

**Stack:** Python, HuggingFace Transformers, PyTorch. **Hardware: 3090 (24GB VRAM) runs the full 8B model unquantized in FP16 (~16GB).** No cloud needed — all local inference.

### 🧪 3. NPC Society: Generative Agents in Roblox

Inspired by Stanford's "Generative Agents" paper (2023, arxiv.org/abs/2304.03442) — 25 AI agents living in a simulated town, forming relationships, spreading gossip, planning their days.

**Prior art:** Virtuals Protocol's Project Westworld (~Oct 2024) already did this: 10 agents with personalities, memory, planning, social dynamics. Study the GAME framework architecture first — then decide whether to replicate, critique, or extend.

**The project:**
- A small Roblox world (village, school, space station) with 5-8 NPCs
- Each NPC has a personality via system prompt + a memory journal (stored as string, summarized and fed back)
- NPCs talk to players AND to each other on a schedule
- NPCs form opinions about players based on interactions ("Player123 was rude to me") and gossip to other NPCs
- Emergent behavior: alliances, conflicts, NPCs warning players about other NPCs

**Key constraints to design around:**
- **5K token context**: System prompt + personality + memory summary + conversation must fit in ~3,750 words. Memory summarization strategy is critical — this IS the ML challenge.
- **17+ maturity restriction**: Extended interactions force Restricted rating. Options: (a) accept small audience, (b) use HttpService + external LLM to bypass, (c) design for short but dense NPC interactions.
- **100 req/sec/experience**: Fine for one player, but 8 NPCs with autonomous conversation schedules + player interactions could hit limits. Needs rate budgeting.

**ML depth:** Memory management under tight context windows, summarization, personality consistency, multi-agent coordination, emergent behavior. These are real research problems — the constraints make them harder and more interesting.

**What's new vs. Westworld:** Westworld used external APIs (not native TextGenerator). She could explore what's possible purely within Roblox's native API constraints — or build a hybrid that's more accessible.

### 📊 4. Roblox Game Success Predictor

Predict game success from observable features (Python project, lives outside Roblox).

**The project:**
- Roblox public APIs give: game name, description, genre tags, thumbnail, player counts, like ratios, update frequency, creator info
- Scrape metadata for top 5,000 games + sample of low-performing games
- Multi-modal model: text features (description NLP) + image features (thumbnail CNN) + tabular features (age, update frequency) → predict concurrent player count
- Feature importance analysis: what actually predicts success?
- Build a "rate my game page" tool for Roblox creators

**ML depth:** Feature engineering, multi-modal learning, regression, real self-curated dataset. Proper data science project with practical output. Most robust and self-contained project on this list — least likely to hit unexpected blockers.

### 🎮 5. Agentic NPCs: AI That *Does Things*, Not Just Talks

NPCs that take actions in the game world based on conversation — open doors, give items, spawn enemies, change weather, track multi-step quest state.

**The project:**
- Use TextGenerator to give NPCs a structured output format (JSON or tagged commands)
- Build an action parser in Lua: conversation → grounded game actions
- Add a planning layer: "Bring me 3 crystals and I'll open the vault" — NPC tracks multi-step quest state via conversation context

**Reality check:** DeterminantAI already ships a plug-and-play agent NPC module with action execution, quest giving, etc. The novelty bar has risen since early 2025. To be interesting, she'd need to go beyond what these SDKs offer — e.g., NPCs that reason about the game state in novel ways, adversarial NPCs that work against each other, or NPC-to-NPC negotiation.

**17+ constraint:** Extended interactions force Restricted rating. For a quick warmup/learning project this is fine; for something her friends play, she'll need short interaction design or HttpService routing.

**ML learning:** Prompt engineering, structured generation, state machines, the gap between language understanding and grounded action (core problem in agentic AI right now).

**Stack:** Pure Roblox Studio + Lua + native TextGenerator. Can start in an afternoon.

## More Ideas (Non-Roblox)

### 6. "Can GPT Do Calc BC?"

Systematically benchmark frontier models on Calc BC problems. Not just "can it solve them" but *where* and *how* they fail. Build a taxonomy of failure modes (algebraic errors? conceptual misunderstanding? correct method but arithmetic mistakes?). She has genuine domain expertise most ML researchers don't. Publishable-quality eval work, timely given the reasoning-model race.

**Novelty sharpening:** Existing math benchmarks (MATH, GSM8K, MathBench, MMLU-math) already test LLMs on math — the novelty is NOT "test LLMs on math problems." It's: (a) **AP Calc BC curriculum-aligned test set** mapping to specific BC topics (integration by parts, Taylor series, polar curves, etc.), not abstract competition math; (b) **qualitative error taxonomy from a student's perspective** — she is simultaneously the domain expert and the target user ("can AI tutor Calc BC students?"); (c) **longitudinal freshness** — as new models release (GPT-5.x, Claude 4, Gemini Ultra), she re-runs the benchmark. The project stays perpetually current.

**Science fair angle:** Co-strongest candidate for GSDSEF 2027 → ISEF alongside #2. "Mathematics" or "Software Design" category. Lowest effort, highest originality. Research question practically writes itself: "Where and how do frontier reasoning models fail on AP Calculus BC?" Quantitative benchmark + qualitative taxonomy. Can extend for STS by adding new models as they release throughout senior year.

### 7. Adversarial Patches IRL

Print physical patches (stickers) that fool an image classifier. Take a MobileNet, find an adversarial perturbation that makes it classify a banana as a toaster. Print it. Stick it on a banana. Film the phone camera misclassifying in real-time. She'll grok how fragile vision models are — and it makes an incredible demo video. Tools: `torchvision` + gradient ascent on the input image.

### 8. AI Dungeon Master

Text-based RPG where an LLM acts as dungeon master. She engineers the prompt system, memory management (context window strategy), and a "consistency checker" — a second model call that validates whether the DM is contradicting its own world. Learns prompt engineering, state management, and the real unsolved problem of LLM coherence. High motivation if she plays any RPGs.

### 9. Predict Her Own Phone Screen Time

Use actual iOS screen time data + calendar + weather + school schedule as features. Train a model to predict tomorrow's screen time. The twist: once it works, she *tries to beat it* — deliberately use phone less than predicted. Turns ML into a self-improvement game. Regression, feature engineering, time series, behavioral science in one project. Underrated idea — personal, unusual, and genuinely teaches data science.

### 10. Few-Shot Plant Disease Detection

Photograph leaves, use a pre-trained vision model + few-shot learning (or fine-tune with 20-50 images per class) to detect plant diseases. Deploy on Raspberry Pi with camera, running inference on cron, sending alerts to phone. End-to-end ML in the physical world.

### 11. Train a Tiny Language Model from Scratch

Karpathy's nanoGPT or llm.c, but on a weird corpus — her own text messages, school newspaper archive, or every Taylor Swift lyric. Small enough to train on a laptop GPU. She'll understand transformers, attention, and tokenization at the implementation level — not the "import transformers" level. With Calc BC she can handle the math.
