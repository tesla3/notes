# Procedural Pre-pretraining: Using Formal Languages & Automata to Improve LLM Reasoning

**Date:** 2025-03-19
**Status:** Research notes

## Summary

A line of research shows that "pre-pretraining" — a lightweight warm-up phase on data generated by formal automata/grammars *before* standard pretraining — dramatically improves LLM training efficiency and reasoning capabilities. Counter-intuitively, pre-pretraining on formal languages works *better* than pre-pretraining on natural language.

## Key Papers

### 1. "Between Circuits and Chomsky: Pre-pretraining on Formal Languages Imparts Linguistic Biases"

- **Authors:** Michael Y. Hu et al.
- **Published:** ACL 2025 (July 2025)
- **Link:** [aclanthology.org/2025.acl-long.478](https://aclanthology.org/2025.acl-long.478.pdf)
- **ArXiv:** [2502.19249](https://arxiv.org/abs/2502.19249)

**Key findings:**

- Pre-pretraining on k-Shuffle Dyck languages (sequences generated by formal automata/grammars) is **more effective token-per-token than natural language** for building inductive biases in LLMs.
- Trained Pythia-1B on 1.63B tokens from C4 — pre-pretraining on formal languages reached the baseline's final loss using only ~1.10B total tokens.
- The term "pre-pre-training" originates here: a warm-up phase on formal language data *before* standard pretraining.
- Uses MAML (model-agnostic meta-learning) to learn initial weights that generalize across formal languages, then transfers to natural language.

### 2. "Procedural Pretraining: Warming Up Language Models with Abstract Data"

- **Authors:** Liangze Jiang, Zachary Shinnick, Anton van den Hengel, Hemanth Saratchandran, Damien Teney
- **Published:** January 2026
- **ArXiv:** [2601.21725](https://arxiv.org/abs/2601.21725)
- **Code:** [github.com/zlshinnick/procedural-pretraining](https://github.com/zlshinnick/procedural-pretraining)

**Key findings:**

- Front-loading just **0.1% procedural tokens** significantly outperforms standard pretraining on NL, code, and math.
- Models reach the same loss with only **55%** (C4), **67%** (CodeParrot), **86%** (DeepMind-Math) of original data.
- Context recall (Needle-in-a-haystack) jumps from **10% → 98%** with Dyck-sequence pre-pretraining.
- Uses k-Dyck languages, cellular automata (Rule 110), stack operations, and other procedural data.
- Attention layers benefit structured domains (code); MLP layers benefit natural language.
- Core thesis: **disentangle knowledge acquisition from reasoning** — let the model learn algorithmic scaffolding first, then semantic content.

### 3. "Transformers Pretrained on Procedural Data Contain Modular Structures for Algorithmic Reasoning"

- **ArXiv:** [2505.22308](https://arxiv.org/abs/2505.22308)

- Examines which architectural components benefit most from procedural pretraining.
- k-Dyck languages and automata-generated sequences create modular structures in small transformers that improve algorithmic reasoning.
- Selective weight perturbation shows tailored pretraining can outperform traditional methods.

### Related: McCoy & Griffiths (2023) — "Prior Distillation"

- **ArXiv:** [2305.14701](https://arxiv.org/abs/2305.14701)
- Formalized the idea of distilling inductive biases into initial weights via meta-learning on formal languages.
- "Prior-trained" networks learn natural language (English) faster, not just formal languages.

## Core Mechanism

The insight is that LLMs learn two entangled things during pretraining:
1. **Knowledge** (facts, relationships, semantics)
2. **Reasoning procedures** (algorithmic manipulation of knowledge)

Standard pretraining conflates these. Pre-pretraining on formal languages isolates (2) — building "algorithmic scaffolding" without semantic shortcuts. This is analogous to how infants learn stacking blocks before higher reasoning.

### Types of procedural data used:

| Data type | What it teaches |
|-----------|----------------|
| k-Dyck languages (balanced brackets) | Recursive/nested structure, counting, context tracking |
| k-Dyck Shuffle (non-nested) | Non-nested dependency tracking |
| Cellular automata (Rule 110) | Local-to-global pattern propagation |
| Stack operations | LIFO memory, sequential state tracking |
| Sorting algorithms | Comparison, ordering |

## Why It Matters

- **Efficiency:** Massive reduction in training data needed (up to 45% less for equivalent performance).
- **Reasoning:** Directly addresses the knowledge-reasoning entanglement problem in LLMs.
- **Practical:** Procedural data is trivially cheap to generate — no web scraping, no human annotation.
- **Theoretical:** Connects to Chomskyan linguistics — formal language priors may approximate the innate "language faculty" hypothesis.

## Counter-intuitive result

Pre-pretraining on **formal** languages works better than pre-pretraining on **natural** language. This suggests the structural regularities of automata-generated strings (recursion, nesting, counting) are more valuable as initial scaffolding than semantic content.
